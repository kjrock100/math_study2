# 0.2.6 Graphs 분석

이 문서는 객체들 사이의 연결 관계를 모델링하는 **그래프(Graphs)**에 대해 다루는 **0.2.6 Graphs** 섹션을 분석하고 설명합니다. 그래프는 이산 수학에서 가장 활용도가 높은 구조 중 하나입니다.

## 그래프의 정의 (Definition of Graphs)

* **정의:** 그래프 $G = (V, E)$는 정점(Vertex)들의 집합 $V$와 간선(Edge)들의 집합 $E$로 구성됩니다.
  * $V$: 공집합이 아닌 정점(또는 노드)의 집합.
  * $E$: 정점 쌍을 연결하는 간선의 집합.

## 그래프의 종류 (Types of Graphs)

1. **무방향 그래프 (Undirected Graph):** 간선에 방향이 없는 그래프입니다. 간선은 비순서쌍 $\{u, v\}$로 표현됩니다.
    * 예: 페이스북 친구 관계 (A와 B가 친구이면, B와 A도 친구).
2. **방향 그래프 (Directed Graph / Digraph):** 간선에 방향이 있는 그래프입니다. 간선은 순서쌍 $(u, v)$로 표현되며, $u$에서 $v$로 가는 화살표로 그립니다.
    * 예: 웹페이지 링크 (A가 B를 링크함), 트위터 팔로우.
3. **가중치 그래프 (Weighted Graph):** 간선에 비용이나 거리 같은 가중치(Weight)가 할당된 그래프입니다.
    * 예: 도시 간의 거리, 네트워크 대역폭.

## 주요 용어 (Key Terminology)

* **차수 (Degree):** 정점에 연결된 간선의 수입니다.
  * 방향 그래프에서는 **진입 차수(In-degree)**와 **진출 차수(Out-degree)**로 나뉩니다.
* **경로 (Path):** 정점들을 연결하는 간선들의 순서열입니다.
* **사이클 (Cycle):** 시작 정점과 끝 정점이 같은 경로입니다.
* **연결성 (Connectivity):** 무방향 그래프에서 모든 정점 쌍 사이에 경로가 존재하면 **연결 그래프(Connected Graph)**라고 합니다.

## 컴퓨터 과학에서의 응용

1. **네트워크 모델링:** 컴퓨터 네트워크, 소셜 네트워크, 도로망 등을 표현합니다.
2. **최단 경로 알고리즘:** 내비게이션 시스템(다익스트라 알고리즘)에서 두 지점 간 가장 빠른 길을 찾을 때 사용됩니다.
3. **웹 검색:** 구글의 페이지랭크(PageRank) 알고리즘은 웹을 거대한 그래프로 보고 페이지의 중요도를 계산합니다.
4. **스케줄링:** 작업 간의 의존성을 그래프로 표현하여 실행 순서를 결정합니다.

## 요약

**0.2.6 Graphs**에서는 복잡한 시스템의 구조를 시각적이고 수학적으로 표현하는 방법을 배웁니다. 그래프 이론은 네트워크 분석, 경로 탐색, 최적화 문제 등 컴퓨터 과학의 난제들을 해결하는 강력한 도구를 제공합니다.
