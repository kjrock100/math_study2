# 0.2.3 Functions 분석

이 문서는 이산 구조의 중요한 구성 요소인 **함수(Functions)**에 대해 다루는 **0.2.3 Functions** 섹션을 분석하고 설명합니다. 함수는 입력과 출력 사이의 관계를 정의하며, 알고리즘의 동작 원리나 복잡도를 설명하는 데 필수적입니다.

## 함수의 정의 (Definition of Functions)

* **정의:** 두 집합 $A$와 $B$에 대하여, $A$의 **각 원소(each element)**가 $B$의 **유일한 원소(exactly one element)**에 대응될 때, 이를 $A$에서 $B$로의 함수라고 합니다.
* **표기:** $f: A \rightarrow B$
* **매핑:** $f(a) = b$일 때, $f$는 $a$를 $b$에 매핑(map)한다고 합니다.

## 주요 용어 (Terminology)

함수 $f: A \rightarrow B$에 대하여:

* **정의역 (Domain):** 집합 $A$. 함수의 입력값들의 집합입니다.
* **공역 (Codomain):** 집합 $B$. 함수의 출력값이 될 수 있는 후보들의 집합입니다.
* **상 (Image):** $f(a) = b$일 때, $b$는 $a$의 상입니다.
* **원상 (Preimage):** $f(a) = b$일 때, $a$는 $b$의 원상입니다.
* **치역 (Range):** 실제로 선택된 공역의 원소들의 집합입니다. $\{f(a) \mid a \in A\}$. 치역은 공역의 부분집합입니다.

## 함수의 종류 (Types of Functions)

### 1. 단사 함수 (Injective / One-to-One)

* **정의:** 서로 다른 입력이 항상 서로 다른 출력을 가지는 함수입니다.
* **조건:** $\forall a, b \in A, (f(a) = f(b) \rightarrow a = b)$
* **특징:** 충돌(Collision)이 발생하지 않습니다. 해시 함수 설계 시 중요한 개념입니다.

### 2. 전사 함수 (Surjective / Onto)

* **정의:** 공역의 모든 원소가 적어도 하나의 정의역 원소에 대응되는 함수입니다.
* **조건:** $\forall b \in B, \exists a \in A (f(a) = b)$
* **특징:** 공역과 치역이 같습니다.

### 3. 전단사 함수 (Bijective / One-to-One Correspondence)

* **정의:** 단사 함수이면서 동시에 전사 함수인 경우입니다.
* **특징:** 정의역과 공역의 원소 개수가 같으며(유한 집합일 경우), 역함수가 존재합니다.

## 역함수와 합성함수

* **역함수 (Inverse Function):** $f: A \rightarrow B$가 전단사 함수일 때, $f^{-1}: B \rightarrow A$가 존재합니다. $f(a)=b$이면 $f^{-1}(b)=a$입니다.
* **합성함수 (Composition):** 두 함수 $g: A \rightarrow B$와 $f: B \rightarrow C$가 있을 때, $(f \circ g)(x) = f(g(x))$입니다.

## 바닥 함수와 천장 함수 (Floor and Ceiling Functions)

이산 수학과 컴퓨터 과학에서 정수 처리를 위해 자주 사용되는 함수입니다.

* **바닥 함수 (Floor):** $\lfloor x \rfloor$는 $x$보다 작거나 같은 가장 큰 정수입니다. (내림)
  * 예: $\lfloor 3.14 \rfloor = 3$, $\lfloor -2.5 \rfloor = -3$
* **천장 함수 (Ceiling):** $\lceil x \rceil$는 $x$보다 크거나 같은 가장 작은 정수입니다. (올림)
  * 예: $\lceil 3.14 \rceil = 4$, $\lceil -2.5 \rceil = -2$

## 컴퓨터 과학에서의 응용

1. **해시 함수 (Hash Functions):** 임의의 크기의 데이터를 고정된 크기의 값으로 매핑합니다. 충돌을 최소화하는 것이 목표입니다.
2. **알고리즘 복잡도:** $O(n)$, $\Omega(n)$, $\Theta(n)$ 표기법은 함수를 사용하여 알고리즘의 성능을 나타냅니다.
3. **프로그래밍 언어:** 프로그래밍에서의 함수(메서드)는 수학적 함수와 유사하지만, 부작용(Side Effect)을 가질 수 있다는 점에서 차이가 있습니다.
4. **데이터 타입 변환:** `int()` 함수는 실수에서 정수로의 매핑(보통 버림)을 수행합니다.

## 요약

**0.2.3 Functions**에서는 입력을 출력으로 변환하는 규칙인 함수를 수학적으로 엄밀하게 정의합니다. 특히 단사, 전사, 전단사의 개념은 데이터 구조의 매핑 효율성을 분석하거나 암호학적 알고리즘을 설계할 때 기초가 되는 중요한 개념입니다.
