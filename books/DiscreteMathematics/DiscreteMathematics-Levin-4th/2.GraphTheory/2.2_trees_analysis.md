# 2.2 Trees 분석

이 문서는 **2.2 Trees** 섹션의 내용을 분석하고 설명합니다. 트리는 그래프 이론에서 가장 중요하고 응용 분야가 넓은 구조 중 하나로, 사이클이 없는 연결 그래프로 정의됩니다.

## 1. 섹션 개요 (Overview)

트리(Tree)는 컴퓨터 과학에서 계층적 데이터(파일 시스템, HTML DOM, 조직도 등)를 표현하거나 효율적인 검색 및 정렬 알고리즘(이진 탐색 트리, 힙)을 구현하는 데 필수적인 자료구조입니다. 이 섹션에서는 트리의 수학적 정의와 성질, 그리고 다양한 종류의 트리를 다룹니다.

## 2. 트리의 정의와 성질 (Properties of Trees)

### 2.1 Definition 2.2.1 (Tree)

그래프 $T$가 **트리(Tree)**라는 것은 $T$가 연결되어 있고(connected) 사이클이 없는(acyclic) 그래프임을 의미합니다.

* **숲 (Forest):** 사이클이 없는 그래프입니다. 숲의 각 연결 성분(Connected Component)은 트리입니다.

### 2.2 주요 성질

정점이 $n$개인 그래프 $T$에 대해 다음 명제들은 모두 동치입니다.

1. $T$는 트리이다.
2. $T$는 사이클이 없고, $n-1$개의 간선을 가진다.
3. $T$는 연결되어 있고, $n-1$개의 간선을 가진다.
4. $T$의 임의의 두 정점 사이에는 **유일한 경로(Unique Path)**가 존재한다.
5. $T$에 간선을 하나라도 추가하면 사이클이 생성된다.
6. $T$에서 간선을 하나라도 제거하면 연결이 끊어진다 (즉, 모든 간선이 다리(Bridge)이다).

## 3. 신장 트리 (Spanning Trees)

### 3.1 정의

그래프 $G$의 **신장 트리**는 $G$의 모든 정점을 포함하는 부분 그래프(Subgraph)이면서 트리인 그래프입니다.

* 모든 연결 그래프는 적어도 하나의 신장 트리를 가집니다.

### 3.2 응용

* **최소 신장 트리 (MST):** 간선에 가중치가 있을 때, 가중치 합이 최소가 되는 신장 트리 (크루스칼 알고리즘, 프림 알고리즘). 네트워크 구축 비용 최소화 등에 사용됩니다.

## 4. 루트 트리 (Rooted Trees)

### 4.1 정의

특정 정점 하나를 **루트(Root)**로 지정하여 계층 구조를 만든 트리입니다. 루트를 지정하면 간선에 부모-자식 관계(방향성)가 생깁니다.

### 4.2 용어

* **부모 (Parent) / 자식 (Child):** 루트 쪽으로 연결된 정점이 부모, 반대쪽이 자식입니다.
* **잎 (Leaf):** 자식이 없는 정점.
* **내부 정점 (Internal Vertex):** 자식이 있는 정점.
* **형제 (Sibling):** 같은 부모를 둔 정점들.
* **조상 (Ancestor) / 자손 (Descendant):** 경로 상에 있는 상위/하위 정점들.
* **높이 (Height) / 깊이 (Depth):** 루트로부터의 거리.

## 5. 학습 포인트

1. **정의의 동치성:** 트리의 여러 정의($n-1$ 간선, 유일한 경로 등)가 서로 같음을 증명할 수 있어야 합니다.
2. **구조적 특징:** 트리는 그래프이면서 동시에 재귀적인 구조를 가집니다. 이를 이용한 귀납적 증명이 자주 사용됩니다.
3. **응용:** 신장 트리를 찾는 알고리즘이나 루트 트리의 순회(Traversal) 방법은 알고리즘 과목의 기초가 됩니다.

## 요약

**2.2 Trees**는 단순해 보이지만 강력한 성질을 가진 구조입니다. "사이클이 없다"는 제약 조건이 "유일한 경로"라는 강력한 성질을 만들어내며, 이는 데이터의 효율적인 관리와 탐색을 가능하게 합니다.
