# 1.1.3 Quantifiers and Predicates 분석

이 문서는 **1.1.3 Quantifiers and Predicates** 섹션의 내용을 분석하고 설명합니다. 이 섹션에서는 단순한 명제 논리(Propositional Logic)의 한계를 넘어, 변수와 범위를 다루는 **술어 논리(Predicate Logic)**의 기초를 다룹니다.

## 1. 술어 (Predicates)

### 1.1 정의

* **술어(Predicate)**는 변수($x, y, \dots$)를 포함하는 문장으로, 변수의 값이 정해지기 전까지는 참/거짓을 판별할 수 없는 문장입니다.
* 변수에 구체적인 값이 대입되면 **명제(Statement)**가 됩니다.

### 1.2 표기법

* 일반적으로 대문자 $P, Q, R$ 등을 사용하여 $P(x)$와 같이 표기합니다.
* 여기서 $P$는 술어의 이름이고, $x$는 변수입니다.

### 1.3 예시

* $P(x): x > 3$
  * $P(2)$는 "$2 > 3$"이므로 **거짓(False)**입니다.
  * $P(4)$는 "$4 > 3$"이므로 **참(True)**입니다.
* $Q(x, y): x + y = 0$ (변수가 여러 개인 경우)

## 2. 한정자 (Quantifiers)

변수에 값을 하나씩 대입하는 대신, 변수의 **범위(Domain)** 전체에 대해 술어의 참/거짓을 이야기할 때 사용합니다.

### 2.1 전칭 한정자 (Universal Quantifier)

* **기호:** $\forall$ (For All)
* **의미:** "모든 ~에 대하여", "임의의 ~에 대하여"
* **형식:** $\forall x P(x)$
* **판별:** 정의역(Domain)에 있는 **모든** $x$에 대해 $P(x)$가 참이어야 전체가 참이 됩니다. 하나라도 거짓이면(반례, Counterexample) 전체는 거짓입니다.

### 2.2 존재 한정자 (Existential Quantifier)

* **기호:** $\exists$ (There Exists)
* **의미:** "어떤 ~가 존재하여", "적어도 하나의 ~에 대하여"
* **형식:** $\exists x P(x)$
* **판별:** 정의역에 있는 $x$ 중 **하나라도** $P(x)$를 만족하면 참이 됩니다. 모두 거짓이어야 전체가 거짓이 됩니다.

## 3. 정의역 (Domain of Discourse)

한정자를 사용할 때는 반드시 변수 $x$가 속한 집합, 즉 **정의역**을 명시해야 합니다. 정의역에 따라 명제의 참/거짓이 달라질 수 있습니다.

* **예시:** $\forall x (x^2 \ge 0)$
  * 정의역이 **실수($\mathbb{R}$)**라면: **참**
  * 정의역이 **복소수($\mathbb{C}$)**라면: **거짓** (예: $i^2 = -1$)

## 4. 한정자의 부정 (Negating Quantifiers)

드모르간의 법칙과 유사하게, 한정자가 포함된 명제를 부정할 때는 한정자의 종류가 바뀌고 술어가 부정됩니다.

1. **전칭 한정자의 부정:** $\neg (\forall x P(x)) \equiv \exists x (\neg P(x))$
    * "모든 $x$가 $P(x)$인 것은 아니다" $\iff$ "$P(x)$가 아닌 $x$가 적어도 하나 존재한다."
2. **존재 한정자의 부정:** $\neg (\exists x P(x)) \equiv \forall x (\neg P(x))$
    * "$P(x)$인 $x$가 존재하지 않는다" $\iff$ "모든 $x$에 대해 $P(x)$가 아니다."

## 5. 컴퓨터 과학과의 연결

* **반복문과 한정자:**
  * `forall`($\forall$): 배열의 모든 요소를 순회하며 조건을 검사하다가, 하나라도 만족하지 않으면 `False`를 반환하는 로직.
  * `exists`($\exists$): 배열을 순회하다가 조건을 만족하는 요소를 하나라도 발견하면 `True`를 반환하는 로직.
* **데이터베이스 쿼리:** SQL의 `ALL`, `ANY`, `EXISTS` 연산자는 술어 논리의 한정자를 직접적으로 구현한 것입니다.

## 요약

**1.1.3 Quantifiers and Predicates**는 명제를 더 정밀하게 표현하는 도구를 제공합니다. 우리는 이제 "모든 소수는 홀수이다"(거짓, 2 때문에)와 같이 집합 전체의 성질을 논리적으로 다룰 수 있게 되었습니다.
